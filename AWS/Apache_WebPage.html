Project Overview

The project demonstrates the creation of two virtual machines, network configuration, SSH connectivity tests between the VMs, private and public IP addresses, security group configuration, Elastic IP assignment, storage expansion using EBS volumes, snapshot creation, AMI image creation, S3 storage test and creation, and Apache web server hosting.

Compute

Two Ubuntu EC2 instances were deployed named VM1 and VM2 respectively.

Both VMs were created using the AWS Free Tier on the t3.micro instance type with 8 GB storage attached to each.

Networking

An Elastic IP (static IP) address was created and assigned VM1 to allow consistent access to the virtual machine without the IP address changing after restart.

A security group was configured to control inbound and outbound traffic.

SSH trust was also configured from VM1 to VM2 to allow for passwordless SSH authentication between the instances.

Storage

An additional EBS volume was created and attached to the VM for extra storage. The additional disk volume was also tested to verify that files could be stored and accessed successfully.

A snapshot of the EBS volume was also created to demonstrate the difference between AWS and VMware snapshot functionality, unlike VMware where snapshots are usually taken of the whole VM, in AWS snapshots are created for individual disk volumes.

Backup / Image

An AMI image was created from VM1 to demonstrate golden image creation within AWS, similarly to in VMware.
